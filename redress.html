Overview

Redress is an interactive narrative game designed to teach students about Japanese American incarceration during WWII — built in partnership with Prof. Lien Tran and Full Spectrum Features as part of a public history research initiative.

The game places players in the role of a library researcher uncovering archival evidence of Executive Order 9066. Rather than presenting history as fact to absorb, Redress makes it experiential — players make choices, navigate characters, and sit with the weight of what they find.

My role spanned the full design-to-implementation pipeline: conceiving the dialogue system, wireframing the UI, and building the interaction layer directly in Twine using SugarCube macros, custom CSS, and JavaScript.

Design Goals

  • Make historical information experiential through player choices and consequences
  • Avoid trauma voyeurism — center resilience, context, and redress instead
  • Design character-driven dialogue that shifts layout and tone per speaker
  • Deliver educational content through gameplay, not exposition
  • Build a system that could scale to additional characters and scenes

Building the Dialogue System

Redress's narrative depended on character-driven dialogue where the portrait, name tag, and layout shift based on who's speaking. I designed and implemented this switching logic in Twine using SugarCube macros, custom CSS, and step-based state management — treating the dialogue system as both a UX pattern and a piece of interaction architecture.

Twine isn't built for this level of transition. I built it anyway — starting from a sketch, ending with a working fade-in/fade-out character switch system that Twine has no native support for.
